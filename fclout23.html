User Guidelines

Alur Kerja OUT FCL BC 23
Penjelasan Flowchart FCL OUT 23
  1. Sistem akan langsung menarik data BC-2.3, jika data diterima maka Status di Sistem berubah menjadi HOLD
  2. Apabila data tidak tertarik secara otomatis maka Customer Servise akan melakukan Proses ON DEMAND, apabila setelah Proses On Demand data berhasil di Tarik maka Status di Sistem akan berubah menjadi HOLD, namun apabila belum tertarik juga maka Customer Servise akan memeriksa dokumen yang dibawa oleh EMKL, apabila data dianggap benar maka Customer Servie akan melakukan input data secara manual, apabila terdapat data yang belum benar maka Customer Service meminta EMKL untuk VERIFIKASI dokumen ke BC Gate, setelah data sudah diverifikasi maka Customer Service akan Input Data Secara Manual dan Status di Sistem akan berubah menjadi HOLD
  3. Setelah itu Customer Service akan memeriksa Data Release P2 & PPC3, apabila semuanya sudah RELEASE maka Petugas Manifest akan mengubah status di Sistem menjadi SET OUTGOING setelah itu input Nopol, Tanggal Keluar & Nomor dan Nama Pengurus
  4. Setelah Petugas Manifest selesai input data maka Petugas Operasional akan mencetak WO Release yang kemudian di serahkan kepada Kerani Lapangan untuk melakukan perintah muat.
  5. Setelah itu Peugas Operasional akan melakukan Foto Depan, samping dan belakang, setelah itu maka EMKL menuju ke BC Gate untuk meminta Stiker BC
  6. Langkah selanjutnya adalah mengambil foto Segel Pelayaran, Segel BC, No Container , kemudian EMKL meminta Release Ke BC, Jika BC setuju untuk RELEASE maka Petugas Surat Jalan akan membuat Surat Jalan, Jika belum maka Petugas akan melakukan Foto Ulang
  7. Kemudian Petugas Surat Jalan membuat Surat Jalan untuk diserahkan ke EMKL yang selanjutnya EMKL akan meminta Tanda Tangan BC dan Stempel Basah.
  8. Setelah Surat sudah di tanda tangai oleh BC maka Supir Tap Kartu Dan Status di Sistem akan berubah menjadi OUT.
Proses On Demand
  • Proses On Demand adalah Proses pemanggilan data apabila data tersebut tidak tertarik secara otomatis oleh sistem, pada saat pengisian No. SPPB, Tanggal SPPB harus ditulis sama persis dengan data yang terdapat di Dokumen, On Demand bisa dilakukan dengan cara:
    1. Buka Menu BC 23 pada Menu TPS Online
    2. Isi No SPPB dan tanggal SPPB kemudian Tekan Call SPPB On Demand
    3. Setelah Data di dapatkan maka petugas status akan berubah menjadi HOLD
Proses Input Manual
  • Proses Pengisian Data secara manual apabila data tidak tertarik dan proses On Demand pun tidak berhasil, dalam melakukan Proses Pengisian manual harus sudah mendapatkan izin dari petugas Bea Cukai, Input Manual bisa dilakukan dengan cara:
    1. Petugas memeriksa Verifikasi Dokumen BC 23, kemudian membawa Dokumen ke BC Gate untuk diperiksa
    2. Setelah mendapat tanda tangan BC pada Dokumen, maka petugas akan input Dokumen yang sudah benar tadi
Proses Set Outgoing
  • Sebelum melakukan Set Outgoing, Petugas harus memeriksa status P2 dan PPC3
  • Jika P2 dan PPC3 statusnya telah menjadi Release, maka Container yang akan keluar bisa langsung di Set Outgoing, dengan cara:
  • Dengan catatan, walaupun salah dari P2 dan PPC3 terkena Hold Container tetap bisa di Set Outgoing dan bisa meminta Release setelah di Set Outgoing.
    1. Buka menu Stored Pada Menu Operasional
    2. Setelah terbuka, pilih salah satu Container yang akan keluar
    3. Tap Gear diatas detail container, lalu pilih Set Outgoing
    4. Masukan No Polisi Mobil Pengangkut, Tanggal estimasi Container akan keluar kemudian Save
Proses P2 Releaseed
  • Jika P2 terkena Hold, Petugas Operasional harus berkoordinasi dengan Petugas P2 untuk meminta Released, dengan cara:
    1. Buka menu Outgoing Pada Menu P2
    2. Setelah terbuka, pilih salah satu Container yang akan di Released
    3. Setelah terbuka periksa nomor dokumen dan isi keterangan apabila diperlukan
    4. Kemudian ganti Status SPPB dari Hold menjadi Released
    Proses Outgoing
    1. Setelah Set Outgoing dan Informasi keluar Container sudah lengkap maka Petugas harus mencetak WO Relase yang kemudian di serahkan kepada Kerani Lapangan untuk melakukan Perintah Muat
    2. Setelah dilakukan proses Muat, kemudian petugas mengambil foto bagian Depan, Samping, dan Belakang Container, dengan cara:
      • Buka Menu Outgoing
      • Tap Gear lalu pilih Container View
      • Pilih Nomor Container yang akan keluar
      • Setelah dipilih, kemudian Tap Gear untuk mengambil masing masing foto
    3. Setelah mengambil foto diatas, EMKL menuju BC Gate untuk meminta Stiker BC 2.3
    4. Petugas Operasional harus mengambil foto Segel Pelayaran, Segel BC, dan Foto Nomor Container
      • Buka Menu Outgoing
      • Pilih Nomor Container yang akan keluar, kemudian Tap Gear untuk mengambil Foto
    5. Setelah mengambil foto kemudian EMKL menuju BC Gate untuk meminta Released, PPC2 akan me-released dengan cara:
      • Buka menu Outgoing Pada Menu PPC2
      • Setelah terbuka, pilih salah satu Container yang akan di Released
      • Setelah terbuka periksa nomor dokumen dan isi keterangan apabila diperlukan
      • Kemudian ganti Status SPPB dari Hold menjadi Released
    6. Setelah mendapat Released dari PPC2 maka petugas akan membuat Surat Jalan yang kemudian akan diserahkan kepada EMKL
    7. Setelah mendapat Surat Jalan, EMKL kemudian meminta Tanda tangan dan Stempel basah kepada BC Gate
    8. Surat di serahkan kepada Supir, apabila Jadi keluar pada tanggal yang di input pertama maka Supir akan Tap Kartu dan Status akan berubah menjadi Out
    9. Jika Mobil tidak jadi keluar pada tanggal Estimasi maka Petugas akan melakukan SET OUTGOING lagi dan akan memasukan Estimasi tanggal baru, dengan cara:
      • Buka Menu Outgoing
      • Pilih Nomor Container yang batal keluar hari ini
      • Tap Gear kemudian Pilih Set Outgoing
      • Setelah di Set Outgoing, Input tanggal Estimasi keluar yang baru
    10. Jika Kartu masih Valid maka Supir bisa Tap kartu dan Status akan berubah menjadi OUT, apabila tidak valid maka Petugas harus menyiapkan Kartu Baru
